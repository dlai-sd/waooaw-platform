Reviewer Notes:  C-065 defines the constitutional structure that makes fully autonomous software
                 development trustworthy — not merely fast. The three mandated separations:

                 1. AUTHOR ≠ REVIEWER
                    The WAOOAW AI Agent — Developer writes code and opens a Pull Request.
                    The PR must be reviewed by one of:
                    a. A human (Sujay for feature quality, Yogesh for constitutional changes)
                    b. A separate GitHub Copilot session invoked with a different office context
                       (e.g., "@copilot review this PR as Enterprise Architect")
                    Self-merge is architecturally prevented by GitHub branch protection.
                    The CODEOWNERS file enforces @dlai-sd (Yogesh) as required reviewer.

                 2. DEPLOYER ≠ DEPLOYMENT CONFIRMER
                    GitHub Actions deploys the code (automated — not the Developer agent).
                    Post-deployment verification is performed by:
                    a. Automated CCT suite (independent of the code that was deployed)
                    b. WAOOAW AI Agent — Platform Operations (monitoring, error rates)
                    The Developer agent does not assess whether its own deployment succeeded.

                 3. SPEC AUTHOR ≠ SPEC APPROVER
                    The Platform IT Expert creates the implementation spec for a GitHub Issue.
                    The spec must be approved by:
                    a. Sujay (for P1 feature work) via GitHub Issue label change
                    b. Yogesh (for P0 work, constitutional changes) via Founder Gate
                    The agent cannot self-approve its own spec.

                 IMPLEMENTATION IN GITHUB:
                 - Branch protection on main: require_pull_request_reviews: 1 approver
                 - CODEOWNERS: @dlai-sd required on all paths (already in place)
                 - Status checks: all CI jobs must pass before merge
                 - Dismiss stale reviews: true (new commits invalidate previous approval)
                 - Restrict merge: no WAOOAW AI Agent can merge without human approval label
