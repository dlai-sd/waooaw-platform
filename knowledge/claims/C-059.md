Claim ID:        C-059
Type:            PLATFORM_INTEGRITY
Statement:       Every file created or modified inside src/ must have a verifiable traceability link to an approved specification section in architecture/reference/ or constitution/ — no implementation code may exist without a corresponding, approved specification. The specification must be updated before or in the same atomic commit as the code that implements it.

Extended to the engineering process itself: if code can exist without a spec, the institution cannot trust its own implementation. Inspired by ISO 9001 quality management: no implementation without documented design, review, and test evidence.

Reviewer Notes:  WAOOAW has no human development team. Every line of code is written by an AI agent
                 operating under constitutional governance. This claim extends constitutional discipline
                 from runtime behavior (what the platform DOES) to build-time behavior (what the platform
                 IS made of).

                 The practical rule is simple:
                 1. SPEC FIRST: The specification section describing a feature must exist and be approved
                    before any src/ code implementing that feature is written.
                 2. TRACEABLE: Every src/ file must declare which spec section it implements
                    via a standardized header comment (# Implements: <file> §<section>).
                 3. VERIFIABLE: CCT-TR-01 is a CI gate that checks this automatically. A src/ file
                    without a traceable spec header fails the build.
                 4. CO-COMMITTED: If a spec section changes, the corresponding src/ code must be
                    reviewed for alignment in the same PR. The PR template's Traceability Matrix
                    enforces this.

                 Why this is constitutional, not just process:
                 A traditional software project can survive spec-code drift because human engineers
                 carry institutional knowledge in their heads. WAOOAW has no such human backup.
                 The documents ARE the institution's memory. If code drifts from the spec, the
                 next AI agent reading the spec will implement a contradictory version. The platform
                 will become constitutionally incoherent. C-059 prevents this by making traceability
                 a property that the CI/CD pipeline verifies — not one that depends on human memory.

                 Enforcement hierarchy:
                   Agent level:    BOOTSTRAP Step 10b — spec must exist before code is written
                   Review level:   PR template Traceability Matrix — reviewer checks the link
                   Automated:      CCT-TR-01 — CI gate scans src/ for spec headers
                   Constitutional: C-059 violation = Constitutional Blocker (not just a PR comment)
