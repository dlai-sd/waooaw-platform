Claim ID:        C-045
Type:            LAW
Statement:       Every instruction given to an AI model that governs how a Digital Professional reasons, decides, or acts is a constitutional document — not an implementation detail. Prompts that define agent behaviour, decision criteria, tone, or constitutional constraints must be versioned, reviewed, and approved under the same governance as Agent Specifications. A prompt change that alters an agent's decision-making is constitutionally equivalent to a Decision Space amendment.
Confidence:      96%
Source:          C-003 (Second Law — authority is licensed; the prompt is the mechanism by which authority is operationalised in the AI model); C-036 (Skills are constitutional units — if the prompt defines how the skill operates, the prompt is part of the skill's constitutional definition); DP-003 (Configuration over Code — prompts are configuration; they must be governed, not hardcoded); GENESIS Part 05 — Agent Definition Protocol requires every behavioural parameter to be specified and approved.
Depends On:      C-003, C-036, DP-003
Produces:        AD-018 (Prompt Versioning and Constitutional Alignment), DP-016 (Prompt-First Execution)
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  This claim addresses a systemic gap in the platform design. Prompts were treated as implementation details — changeable by any developer without governance. But a prompt that tells an Instagram skill "always present clinical claims conservatively" vs "always present clinical claims boldly" produces fundamentally different agent behaviour with different constitutional implications. Changing a prompt without governance is a constitutional violation — it changes the agent's Decision Space operationalisation without a Decision Space amendment. The practical consequence: the Prompt Library is a governance artifact, not a code file. Every prompt has a version, a reviewer, and a constitutional basis citation. Prompts that change how the agent decides (not just phrasing) require EA review. Prompts that change what the agent can do require Founder approval (same as a Decision Space amendment).
