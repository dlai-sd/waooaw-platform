Claim ID:        C-073
Type:            OBLIGATION + PLATFORM_INTEGRITY
Statement:       Every function, class, or module in the WAOOAW platform that implements or enforces a constitutional principle must carry a machine-readable annotation declaring: (1) the constitutional claim(s) it implements, (2) the IB item that authorized its creation, (3) the component specification file that describes its contract. This annotation must be in the source code itself — not only in commit messages or documentation. The platform must be able to answer "which code implements C-043?" by scanning source files without reading Git history. When a constitutional claim is amended, CI must identify all source files annotated with that claim ID and surface them to the reviewing agent. This bidirectional traceability closes the full chain from constitution → running code → audit evidence, and makes the platform genuinely self-documenting.

Reviewer Notes:  WHY COMMIT MESSAGES ARE INSUFFICIENT:

                 C-059 requires "IB: IB-009" in every commit message. This is correct
                 but incomplete. Consider:
                 - A file has 30 functions. The commit says "IB-009, C-041". Which
                   of the 30 functions implements C-041? You cannot know from the commit.
                 - C-041 is amended to add a new prohibited tool category. The developer
                   must grep the codebase by memory. If they miss a function, the amendment
                   is silently incomplete.
                 - Six months from now, a new agent joins. It reads C-041. It wants to
                   know where C-041 is enforced in code. It cannot find out without reading
                   every file in src/.

                 C-073 fixes this with function-level annotations:
                 - Every evaluator that enforces C-041 has @constitutional_claim("C-041")
                 - CI can scan and produce: "C-041 is enforced in: C041Evaluator.cs,
                   test_cct_ce_01.py, audit_records (constitutional_basis column)"
                 - When C-041 is amended: CI warns "These files are tagged C-041 — review"
                 - An agent can navigate: C-041 → C041Evaluator.cs → CCT-CE-01 → audit_records

                 THE COMPLETE BIDIRECTIONAL MAP:
                 Forward:  Constitution → Claim → IB item → Spec file → Source file → CCT → Audit record
                 Backward: Audit record → constitutional_basis → Claim → Source file → Spec file → IB item
                 Amendment: Amend C-043 → CI scan → "C-043 tagged in: C043Evaluator.cs,
                             test_cct_ce_02.py, TrustLedger.cs (budget ceiling formula)"
