Claim ID:        C-086
Type:            OBLIGATION + PLATFORM_INTEGRITY
Statement:       Before any LLM-based code generation task is executed for the first time,
                 a Sprint Task Simulation (SIM-PL-002 format) MUST exist for that task type
                 with a recorded PASS verdict. The simulation must prove — with controlled
                 inputs and verifiable outputs — that the task's dependency structure is
                 correct, the generated code will compile, and the constitutional checks are
                 adequate. No first-attempt LLM call for a new task type is constitutionally
                 authorized without a prior simulation PASS. The violation is not a
                 recommendation to simulate. It is a hard gate: code generation without
                 prior simulation = constitutional violation equivalent to Evidence First
                 bypass (C-023). The pipeline MAY NOT call the LLM for WC-NNN-NN if no
                 SIM-PL-002 for WC-NNN-NN exists in simulation/ with verdict: PASS.

                 Scope: applies to every new sprint task type the first time it appears
                 in TASK_HANDLERS. Does not require re-simulation for repeated runs of
                 the same task with the same dependency structure. Does require re-simulation
                 when the task's dependency structure changes (new files added, new layers).

Confidence:      99%
Source:          C-071 (Quality Obligation — quality gates must be automated and non-negotiable);
                 C-059 (Traceability — every LLM call must trace to a verified spec and
                 a verified execution approach, not just the spec);
                 C-082 (Build Validation — generated code must compile; simulation proves
                 the compilation approach before the LLM call, not after);
                 IB-020 evidence: WC-012 ran 7+ times across multiple sessions, discovering
                 the same namespace inconsistency in multi-layer generation repeatedly.
                 Each run consumed 48,000 tokens and 20+ minutes. A 2-hour simulation
                 before the first run would have prevented all subsequent failures.
                 The cost of skipping simulation is exponential. The cost of running it is fixed.

Depends On:      C-071 (Quality Obligation), C-059 (Traceability), C-082 (Build Validation),
                 C-023 (Evidence First — simulation result IS the evidence that the approach

Reviewer Notes:  WHY THIS IS A CONSTITUTIONAL OBLIGATION AND NOT JUST A BEST PRACTICE:

                 A best practice is followed when convenient. A constitutional obligation is
                 enforced by the pipeline regardless of schedule pressure. WC-012 was under
                 no artificial schedule pressure — we still skipped simulation. The pattern
                 will repeat for every sprint unless the pipeline physically cannot invoke
                 the LLM without a prior simulation PASS.

                 THE THREE FAILURES THAT PROVE THIS IS NECESSARY:
                 - WC011: 7+ failed runs before catching pipeline infrastructure issues.
                   SIM-PL-001 was created AFTER these failures. It now prevents them.
                 - WC012-01: 3 failed runs with different hallucinations (AddGrpcClient-
                   Instrumentation, ActionDecision.ActionDecisionDeny, wrong proto fields).
                   A simulation would have revealed: "the API methods Claude will use don't
                   exist — use deterministic template instead." Zero LLM calls wasted.
                 - WC012-03: 3 failed runs with namespace consistency failures across
                   interdependent files. A simulation would have revealed: "single-LLM-call
                   approach cannot guarantee namespace consistency across Data/ + Services/ +
                   Tests/ — dependency-ordered sub-task execution required."

                 RELATIONSHIP TO C-023 (Evidence First):
                 C-023 says: write evidence before returning success.
                 C-086 says: prove the approach compiles before writing any code.
                 C-023 protects the customer from unrecorded actions.
                 C-086 protects the platform from unproven generation approaches.
                 Both are Evidence First — at different levels of the value chain.

                 THE SIMULATION IS NOT A TEST OF THE OUTPUT:
                 C-076 (coverage) tests what was generated. C-086 tests the APPROACH
                 before generation. They are complementary, not overlapping:
                 - Simulation (C-086): "will the namespace structure work?"
                 - CCTs (C-076): "does the generated code implement the claim correctly?"
                 Neither can replace the other.

                 WHAT CHANGES IN THE PIPELINE:
                 The pre-flight job gains a new check: before building the sprint index,
                 verify that simulation/SIM-PL-002-{task_id}.md exists with "Verdict: PASS".
                 If not: HALT. Reason: "No simulation for {task_id}. C-086 requires simulation
                 PASS before first LLM call. Create simulation first."
                 This is identical to the autonomous_halt check — it is machine-enforced,
                 not human-remembered.
