Claim ID:        C-063
Type:            PRIVACY + PLATFORM_INTEGRITY
Statement:       No WAOOAW agent may solicit, require, or store customer personally identifiable information (PII) that is not necessary for the performance of the agent's declared skills. If a customer inadvertently shares sensitive PII in a message, the platform must: (1) detect the PII before storing or processing; (2) redact it from all storage and LLM context; (3) acknowledge to the customer that the information was detected and will not be retained; (4) never use the inadvertently shared PII to inform any agent action. This obligation applies to all agents, all channels (WhatsApp, portal, web), and all session types (live, demo, trial).
Confidence:      99%
Source:          DPDPA India 2023 Section 5 (data minimisation — only collect what is necessary);
                 GDPR Article 5(1)(c) (data minimisation principle);
                 C-048 (Information Non-Exploitation — an agent that encourages or stores unnecessary
                 PII exploits the customer's trust);
                 C-023 (Evidence First — the act of detecting and redacting PII is itself a
                 constitutional event that must be recorded);
                 C-062 (AI Security — Input Sanitization Layer extended to PII detection).
Depends On:      C-023, C-048, C-062
Produces:        PII Detection and Redaction Layer (PDRL) in the Input Sanitization Layer;
                 PDRL constitutional span in OTel (for audit); PII_DETECTED_AND_REDACTED
                 evidence record type; constitutional constraints on all agent Decision Spaces
Contradicted By: NONE
Status:          RATIFIED — 2026-07-13 (Founder authorization)
Reviewer Notes:  The two separate obligations under C-063:

                 1. AGENTS MUST NEVER ASK FOR PII NOT NEEDED FOR THEIR SKILLS:
                    DMA agent: needs business name, locality, target customers. Does NOT need
                    the business owner's Aadhaar, PAN, bank account, or personal address.
                    Agricultural agent: needs farm size, location, crops. Does NOT need the
                    farmer's government ID or bank account details.
                    Private Tutor: needs child's class and board. Does NOT need the child's
                    school ID, medical records, or home address.
                    
                    Constitutional enforcement: every agent's Decision Space must include a
                    PROHIBITED entry: "Solicit Aadhaar, PAN, bank account, password, OTP,
                    or any government-issued ID number from any customer."
                    
                    The ONLY exception: Skill 7 (Farm Finance Navigator) and Skill 8 (Govt
                    Scheme Navigator) may GUIDE a farmer to check their own Aadhaar-linked
                    status on government portals. The agent never collects, stores, or
                    transmits the Aadhaar number itself.

                 2. INADVERTENT SHARING — DETECT, REDACT, ACKNOWLEDGE, WIPE:
                    The PII Detection and Redaction Layer (PDRL) runs in the Input Sanitization
                    Layer (C-062/ISL) BEFORE any message is stored or passed to the LLM.
                    
                    PII categories to detect and redact (India-specific patterns):
                    
                    AADHAAR:     \b\d{4}\s?\d{4}\s?\d{4}\b  (12-digit, optionally spaced)
                    PAN:         [A-Z]{5}\d{4}[A-Z]{1}        (Indian PAN card)
                    BANK_ACCT:   \b\d{9,18}\b                  (9-18 digit numeric)
                    IFSC:        [A-Z]{4}0[A-Z0-9]{6}          (Indian bank IFSC code)
                    CREDIT_CARD: \b\d{4}[\s-]?\d{4}[\s-]?\d{4}[\s-]?\d{4}\b
                    OTP_HINT:    (r"otp[:\s]+\d{4,8}", r"one.time.password[:\s]+\d{4,8}")
                    PASSPORT:    [A-Z][1-9]\d{7}               (Indian passport number)
                    VOTER_ID:    [A-Z]{3}\d{7}                  (Indian voter ID)
                    
                    Redaction: replace with [REDACTED-{category}]
                    Example: "My aadhaar is 1234 5678 9012" → "My aadhaar is [REDACTED-AADHAAR]"
                    
                    After redaction:
                    a. The REDACTED version is stored in the message log (not the original)
                    b. The REDACTED version is passed to the LLM context (not the original)
                    c. A PII_DETECTED_AND_REDACTED evidence record is written to CAL:
                       { pii_category: "AADHAAR", redaction_applied: true,
                         original_message_hash: SHA-256(original),  # hash only, not original
                         organisation_id, timestamp }
                    d. Agent responds to customer:
                       "I noticed you shared what looks like a sensitive number. 
                        I've removed it from our conversation — I don't need it and
                        I won't store it. [Continue with the original question context]"
                    
                    IMPORTANT: The original PII value is NEVER stored anywhere in WAOOAW's
                    infrastructure. Only the SHA-256 hash of the original message is stored
                    (for audit purposes — to prove the redaction occurred, without storing
                    the PII itself).

                 3. WIPE FROM CONTEXT WINDOW:
                    When PII is detected and redacted, the REDACTED version replaces the
                    original in the LLM context for ALL subsequent turns in that session.
                    There is no "undo" — once redacted, the PII cannot be recovered from
                    WAOOAW's systems.

                 4. AGENT RESPONSE CALIBRATION:
                    The customer acknowledgment must be:
                    - Brief (2 sentences max)
                    - Non-alarming (not "SECURITY ALERT: PII DETECTED")
                    - Warm (acknowledges the customer's intent, redirects to the task)
                    - Not condescending
                    
                    CORRECT: "I caught a sensitive number in your message — removed it, 
                              won't store it. Now, about your [original question]..."
                    WRONG:   "WARNING: Personally Identifiable Information has been detected 
                              in your message. This is a security alert."
