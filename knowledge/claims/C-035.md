Claim ID:        C-035
Type:            LAW
Statement:       All Minimum Viable Institution professional scenarios must run on a single shared runtime codebase with zero runtime code changes — scenario differentiation is achieved exclusively through configuration and Decision Space parameters, never through conditional code branching per professional type.
Confidence:      100%
Source:          ORGANIZATION.md Office 10 — Runtime Universality Test (derived from GENESIS): "A Dentist hiring a Digital Marketing Professional, a Trader hiring a Trading Professional, a Lawyer hiring a Legal Professional, and a Doctor hiring a Healthcare Professional must all run on the same runtime codebase with zero runtime code changes — only configuration and Decision Space parameters differ."
Depends On:      C-030 (Decision Space as architectural primitive — the reason differentiation is through Decision Space parameters)
Produces:        NONE (this is the most important single architectural constraint for Professional Runtime design)
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  Produced in response to R-001 Gap 3. This is a LAW-type claim because it is directly stated in GENESIS as an absolute requirement (the Runtime Universality Test). Its architectural consequence is profound: the Professional Runtime may not contain `if professionalType == TRADING then ... else if professionalType == MARKETING then ...` logic. All professional-specific behavior must flow through Decision Space parameters. Any architecture that requires code changes per professional type has failed this test. This claim is the constitutional authorization for the configuration-over-code design principle.
