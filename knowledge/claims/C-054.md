Claim ID:        C-054
Type:            LAW
Statement:       Every Digital Professional agent that exposes more than one Skill to its customers MUST implement intelligent, request-time Skill routing — the ability to classify any customer message or request against the agent's active Skill capability manifests, identify the primary Skill and any contributing Skills, and dispatch the request to the correct execution context with the correct RAG sources, MCP tools, and Decision Space constraints. When a single request spans multiple Skills, the agent must orchestrate those Skills in dependency order, pass outputs between them correctly, and present one coherent professional response — not multiple disconnected outputs. Every Skill must declare a Skill Capability Manifest (SCM) that the Skill Intelligence Router can reason over. An agent that lacks Skill routing forces the customer to know which tool to use; a professional knows their own capabilities and selects the right one. A multi-skill agent without intelligent routing is not a digital professional — it is a labelled menu.

Reviewer Notes:  C-054 closes the most commercially significant gap in the current agent architecture. A DMA agent with 14 Skills that forces Dr. Mehta to know whether to click "Content Strategy" or "Instagram Marketing" to get a Diwali campaign is not a digital marketing professional — it is a complex menu. C-054 makes the following architectural requirements mandatory: (1) Every Skill declares a Skill Capability Manifest (SCM) — a machine-readable contract of what request types it can serve, what it produces, what it needs, and what Skills it collaborates with. (2) The Skill Intelligence Router (SIR) runs at LOCAL tier cost before the Execution Loop REASON step — it is classification, not reasoning, and adds ₹0 to the execution cost. (3) Multi-skill responses are orchestrated by the Skill Collaboration Orchestrator (SCO), which manages dependency ordering and data handoffs. (4) The SIR's gap_detected flag is the constitutional trigger for the Agent Skill Proposal Governance Loop (Section 3.20) — an unserved customer intent, observed enough times across enough customers, produces an evidence-grounded business case for a new Skill. Constitutional note: the SIR operates on ACTIVE skills only. Skills the Strategic Cognition Layer (C-050) has deactivated are invisible to the SIR. The SIR and the Strategic Cognition Layer operate at different cadences and answer different questions: C-050 asks "which Skills should be in the portfolio?"; C-054 asks "which active Skill(s) serve this request right now?". Both are mandatory and neither replaces the other.
