Reviewer Notes:  WHY THIS IS CONSTITUTIONAL AND NOT JUST TOOLING:

                 Package version selection is an architectural decision, not a coding decision.
                 The choice of Grpc.AspNetCore 2.67.0 vs 2.65.0, or EF Core 9.0.1 vs 9.0.0,
                 can silently change runtime behaviour, security exposure, and constitutional
                 compliance (e.g. a package with a CVE may violate C-062 Security Obligation).

                 An autonomous agent CANNOT make architectural decisions (C-066 Tier 3 boundary).
                 Allowing the agent to choose package versions is therefore a constitutional
                 category error — it is asking the Implementation hat to perform EA work.

                 The reference dotfiles/ pattern resolves this:
                 - EA maintains the reference files (architectural decision authority)
                 - Implementation agent copies verbatim (implementation authority)
                 - Constitutional separation is preserved (C-065, C-066)

                 MOBILE EXTENSION:
                 When a mobile application is added to the platform, a pubspec.yaml reference
                 file (Flutter/Dart) MUST be created in architecture/reference/dotfiles/ and
                 approved by EA before any mobile scaffold task is authorized. The same
                 "copy verbatim" instruction applies. Mobile dependencies carry additional
                 risk: app store compliance (Google Play / App Store security requirements)
                 depends on package versions — a non-compliant dependency can block release.
