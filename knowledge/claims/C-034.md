Claim ID:        C-034
Type:            ARCHITECTURAL_IMPLICATION
Statement:       The employment lifecycle must support at minimum four distinct states — Evaluation (pre-contract, rights visible), Active (contracted and executing), Suspended (paused with evidence preserved), and Terminated (concluded with data portable) — each with distinct constitutional transitions and evidence requirements.
Confidence:      90%
Source:          Derives from C-007 (evidence immutability — evidence must be preserved during Suspended state), C-009 (CP-001 — Evaluation is a constitutional state requiring rights visibility), C-015 (HYPOTHESIS: Suspension is distinct from Termination, CD-011). GENESIS Acceptance Scenarios imply the Active state. Constitution Article IX (Customer Rights — right to terminate immediately) implies Terminated.
Depends On:      C-007, C-009, C-015
Produces:        NONE (this is the core domain model target for the Enterprise Architect)
Contradicted By: NONE
Status:          DRAFT
Reviewer Notes:  Produced in response to R-001 Gap 2. Confidence is 90% (not 95%+) because CD-011 (Suspension) is still under deliberation — if Suspension is rejected as a required state, the lifecycle reduces to Evaluation → Active → Terminated. However, C-009 (Evaluation) and Terminated are constitutionally unambiguous. The architectural recommendation: design the employment state machine to support all four states now; Suspension costs nothing to include and avoids a data architecture redesign if CD-011 is ratified. The evidence state machine (C-028) handles per-action evidence. This claim covers the employment relationship lifecycle — a different and higher-level concern.
