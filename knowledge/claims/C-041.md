Claim ID:        C-041
Type:            LAW
Statement:       Every external tool call made by a Digital Professional — posting to Instagram, executing a trade, sending a WhatsApp message — must be explicitly authorized within the professional's Decision Space before the AI Runtime may invoke the corresponding tool; unauthorized tool calls are a constitutional violation equivalent to unauthorized action execution.
Confidence:      97%
Source:          Derives from C-003 (Second Law — authority is licensed, not assumed; a tool call IS an action); C-030 (Decision Space as primitive — the Decision Space defines what actions the professional may take, and tool calls are the mechanism of action); Constitution Article III (Second Law applies to all professional actions, not only human-visible ones); GENESIS "Configuration over Code" — tool authorization is configuration, not code
Depends On:      C-003, C-030
Produces:        NONE
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  This claim prevents a subtle but critical vulnerability: an AI model that invokes an external API (Instagram posting, trade execution, WhatsApp broadcast) that was not explicitly listed in the customer's authorized_actions. The Constitutional Engine's ValidateAction must be called before every tool invocation, not only before human-visible outputs. The Tool Registry in AI Runtime must enforce "default deny" — if a tool is not in decision_space.authorized_tools, the call is rejected before it reaches the external API. This is the MCP integration's constitutional constraint: MCP makes it easy to add tools; C-041 ensures every tool addition requires explicit customer authorization.
