Claim ID:        C-027
Type:            ARCHITECTURAL_IMPLICATION
Statement:       The Constitutional Audit Ledger must be implemented as an append-only table at the database level — no DELETE, UPDATE, or TRUNCATE SQL operations may be applied to committed records by any database user, including the application service account.
Confidence:      98%
Source:          Derives directly from C-007 (ledger immutability — Constitutional Floor), Constitution Article XI. Confirmed in ADR-011 (Database Migration Strategy — no destructive migrations on constitutional schema).
Depends On:      C-007
Produces:        NONE
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  This is the most direct architectural translation of a Constitutional Floor. "No evidence in any ledger may be retroactively modified or deleted" (Article XI) maps precisely to: the database schema must use a PostgreSQL trigger or privilege restriction to prevent non-append operations. The application service account for the Constitutional Engine must have INSERT on the audit ledger and SELECT on the audit ledger — not UPDATE, not DELETE. This is not a convention; it is a database-level permission enforcement. Any EF Core migration that includes an UPDATE or DELETE on the constitutional schema must be blocked at code review.
