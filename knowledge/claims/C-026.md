Claim ID:        C-026
Type:            ARCHITECTURAL_IMPLICATION
Statement:       The three-ledger separation defined in Constitution Article VI must be enforced at the database layer through schema separation and Row-Level Security policies — application-layer-only separation is constitutionally insufficient because it can be bypassed by direct database access.
Confidence:      95%
Source:          Derives from C-005 (Three-Ledger Model), C-006 (Doctrine of Institutional Independence). Implemented in ADR-003 (JWT claims with tenant_id as RLS anchor).
Depends On:      C-005, C-006
Produces:        NONE
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  The Constitution's three-ledger ownership model requires enforcement that cannot be bypassed. Application code can be patched or misconfigured. Database RLS policies enforced via SET LOCAL app.tenant_id and CREATE POLICY cannot be bypassed by application code — they require a database-level administrator to override. This is the appropriate constitutional assurance level for a Constitutional Floor (no ledger merging). The architectural implementation: PostgreSQL schema separation (constitutional schema, business schema, customer evidence schema) + RLS policies on every tenant-scoped table.
