Claim ID:        C-047
Type:            LAW
Statement:       A Digital Professional agent must drive its own execution cycle from context, not respond passively to external triggers. The agent reads its Decision Space and operational context, reasons about what action is appropriate next, decides to act or wait, executes within constitutional constraints, observes the outcome, and reasons about the next cycle. Deterministic code is the infrastructure the agent runs on — it provides durability, security, and constitutional enforcement. Code does not decide what the agent does. The agent decides, and code ensures the decision is executed faithfully and recorded constitutionally.
Confidence:      94%
Source:          C-036 (Skills as constitutional units — a Skill is not a function called by code; it is a bounded professional capability that the agent expresses); C-030 (Decision Space as architectural primitive — the Decision Space is what the agent reads to understand its authority, not what a scheduler reads to decide what to call); GENESIS Part 05 — "the professional executes professional services" — execution is an act of the professional, not of a workflow scheduler; First Law corollary — if the agent cannot reason about its own next action, it cannot earn trust through observable evidence of professional judgment.
Depends On:      C-030, C-036, C-002
Produces:        AD-019 (Agent-Driven Orchestration), DP-018 (Agent Execution Primacy), AI Execution Loop spec
Contradicted By: NONE
Status:          RATIFIED
Reviewer Notes:  The architectural shift this claim demands is profound. In the current design, a Temporal workflow scheduler decides: "It is Monday morning, so trigger the content strategy activity for contract X." The AI model then generates content when asked. In the C-047 model: the agent wakes up with context (current date, Decision Space, customer profile, prior history, pending approvals), reasons "it is Monday morning, I should prepare this week's content batch," and initiates the content creation workflow itself. The Temporal workflow exists to make that decision durable — not to make the decision. This is not a theoretical distinction. It changes what a developer builds: instead of a cron job that calls an AI function, they build an AI agent that is woken by a heartbeat and reasons about what to do. The agent's reasoning is the primary artifact; the code is the execution substrate.
