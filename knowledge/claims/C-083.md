Statement:       Every WAOOAW agent — whether an autonomous sprint runner, a customer-facing

                 at each significant action boundary. This is the Emit-Transport-Listen
                 obligation. It is distinct from Evidence First (C-023): Evidence First requires
                 writing a permanent record before returning success; C-083 requires emitting a
                 transient, structured, observable signal AT EACH BOUNDARY so that downstream
                 systems, stewards, and customers can determine the agent's current operational
                 state without inspecting internal state or waiting for workflow completion.

                 The three parts of this obligation:

                 EMIT: At each significant action boundary, the agent MUST produce a structured
                 signal object containing: the action type, the agent/session/contract identity,
                 the current step in the workflow, the outcome (STARTED / SUCCESS / FAILURE /
                 ESCALATION), and enough context for a downstream observer to understand what
                 happened. "Significant action boundary" means: session start, each tool call
                 initiation, each tool call result, each LLM inference, each step failure, each
                 escalation, and session end.

                 TRANSPORT: The signal MUST be written to a shared, readable surface accessible
                 to downstream consumers without requiring access to the agent's internal state
                 or its execution branch. For the sprint pipeline: GitHub Actions artifacts.
                 For customer-facing agents: the Constitutional Audit Ledger streaming table
                 (to be specified in the Professional Runtime architecture).

                 LISTEN: Platform services with a constitutional mandate to observe agent
                 behaviour (Self-Improvement Analyst C-069, Constitutional Monitor, Steward
                 Assistant) MUST subscribe to and process these signals. Downstream services
                 that receive signals MUST NOT re-query the source agent for state already
                 emitted in a signal.

                 VIOLATION DEFINITION: An agent that completes a multi-step workflow and
                 emits only a final outcome record is in violation of C-083. Recording the
                 outcome (C-023) is necessary but insufficient. The customer, the steward,
                 and the platform must be able to observe the agent's state at any point
                 during execution — not only after it concludes.

Reviewer Notes:  WHY THIS IS A NEW CLAIM AND NOT AN EXTENSION OF C-023:

                 C-023 says: write a permanent Evidence Ledger record before returning success.
                 C-083 says: emit a transient boundary signal at every action step.

                 They are different obligations on different surfaces:
                 - Evidence Ledger: permanent, append-only, court-of-record (C-027)
                 - Boundary signals: ephemeral, streaming, operational nervous system

                 A doctor's obligation to C-023 is to write in the chart before the consultation
                 ends. The obligation under C-083 is to call out what they are observing AS they
                 examine the patient — so the nurse, the specialist, and the patient can all hear
                 and react in real time.

                 THE SPRINT PIPELINE PROOF:
                 Before C-083: the runner executed 4 tasks, wrote evidence to a branch, opened
                 a PR. The Constitutional Monitor ran 20 minutes later and read nothing because
                 the evidence was on a branch it couldn't access. 9 false spec-gap issues were
                 created. A human session spent 3 hours doing what the monitor should have done.
                 After C-083: the runner emits monitor-signal.json at each task boundary.
                 The monitor downloads it. Classification is instant, correct, and autonomous.

                 THE CUSTOMER PROOF:
                 A Dental Marketing Agent without C-083: runs a campaign, fails at step 1
                 (CRM), continues to steps 2 and 3, sends blank emails to Dr. Mehta's
                 patients, records the outcome in the Evidence Ledger. Dr. Mehta learns what
                 happened after the damage. C-023 is satisfied. The customer is harmed.
                 A Dental Marketing Agent with C-083: emits TOOL_CALL_FAILED at the CRM step.
                 The downstream steps receive a failure signal. They halt (C-084: Step
                 Dependency Ordering). The Self-Improvement Analyst is notified. Dr. Mehta's
                 dashboard shows a real-time alert. No emails go out. C-023 and C-083 are both
                 satisfied. The customer is protected.

                 RELATIONSHIP TO C-069:
                 C-069 requires the platform to raise improvement proposals when degradation
                 is detected. C-083 makes degradation detectable in real time. Without C-083,
                 C-069 can only look backward at completed sessions. With C-083, C-069 can
                 detect degradation during active sessions and intervene before harm occurs.

                 LLM IMPLICATION:
                 The Self-Improvement Analyst and the Constitutional Monitor must use LLM
                 reasoning to interpret boundary signals — not just rule-based counters.
                 A rule counts escalations. An LLM understands WHY the escalations happened
                 and proposes specific remediation. Any platform classification service
                 operating on agent signals without LLM reasoning is constitutionally
                 insufficient — it will miss novel failure patterns and propose generic fixes.
